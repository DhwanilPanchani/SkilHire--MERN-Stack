# SkilHire: A MERN Stack Job Hiring Platform

## Description
**SkilHire** is a fully responsive web application designed as a job hiring platform. Built using the MERN stack (MongoDB, Express.js, React.js, Node.js), the application provides a user-friendly interface for job seekers and employers. It offers secure login, role-specific functionalities, and seamless data management.

---

## Features
- **Mobile UI Friendly/Responsive Design**: Ensures accessibility across all devices.
- **Roles**: 
  - **Admin**: Manages the platform, users, and job listings.
  - **Users (Employers and Job Seekers)**: Employers can post jobs, and job seekers can apply for jobs.
- **Authentication**:
  - Login with password encryption using `bcrypt`.
  - Session management for enhanced security.
- **Pages**:
  - Home
  - Login/Register
  - Dashboard (Admin and User-specific)
  - Job Listings
  - Job Details
  - Profile Management
  - Settings
  - Contact Us
- **Transactions/Flows**:
  - Admin: Manage job listings, users, and platform analytics.
  - User: View and apply for jobs, manage profiles, and track application status.
- **Data Management**:
  - CRUD operations for user profiles and job listings.
  - Data consistency and validation.
- **Frontend**:
  - Designed with **HTML**, **CSS**, and **Bootstrap**.
  - Additional third-party libraries for enhanced UI/UX.
  - **React.js** for dynamic rendering and state management.
- **Backend**:
  - Developed using **Node.js** and **Express.js**.
  - RESTful API architecture for scalable and maintainable endpoints.
- **Database**:
  - **MongoDB** for seamless NoSQL data storage.
- **External Integrations**:
  - Payment gateway API for e-commerce functionality (if applicable).
  - Other APIs for additional features (e.g., email notifications, geolocation, etc.).
- **Hosting**: Hosted on platforms like Heroku for public access.
- **Version Control**:
  - Git branching strategy for development and feature enhancements.
  - Functionality-wise commits with a clear README and `.gitignore`.
